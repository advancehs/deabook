    10. 随机非参数数据包络分析¶

    在效率分析领域，虽然尚未形成统一的理论框架，但数据包络分析（DEA）和随机前沿分析（SFA）已发展成为两种具有互补特征的主流方法。这两种方法论在处理效率评估问题时呈现出不同的技术路径。方法论的选择往往取决于研究数据的特征和具体问题的需求。

    数据包络分析（DEA，Farrell 1957；Charnes et al. 1978）1 2是一种公理的、数学编程的效率分析方法。与计量经济学和基于回归的工具相比，DEA的主要优势在于它对前沿的非参数化处理，建立在生产理论的公理之上，如自由处置性（单调性）、凸性（凹性）和规模收益不变（同质性）。DEA并不假定边界或无效率分布的任何特定函数形式。它的直接的、以数据为导向的方法有助于将效率分析的结果传达给决策者。然而，DEA的主要缺点是它把所有偏离前沿的情况都归结为无效率。这往往是一个非常强的假设。

    SFA的主要优势在于其对偏离前沿的概率建模，这些偏离被分解为一个非负的无效率项和一个特异性误差项，该误差项考虑了一些遗漏的因素，如企业及其经营环境的非观察异质性、测量和数据处理的随机误差、规格误差和其他噪音来源。与DEA相比，SFA采用了参数化的回归技术，这需要事先对前沿的函数形式和无效率分布进行规范。由于经济理论很少证明一个特定的函数形式是合理的，所以经常使用灵活的函数形式，如translog。然而，灵活的函数形式往往违反了生产理论的公理，而强加这些公理则会降低灵活性。

    鉴于不同方法论之间的界限与区别，凸非参数最小二乘法（Convex Nonparametric Least Squares, CNLS）的提出（Kuosmanen，2008）3开创了新的研究路径。CNLS通过二次规划方法估计满足单调性与凸性约束的非参数前沿，既继承了DEA的公理化建模优势，又融合了SFA的统计推断框架。基于CNLS的拓展研究（Kuosmanen & Johnson 2010；Kuosmanen & Kortelainen 2012）4 5 最终形成了统一的分析框架——随机非参数数据包络（Stochastic Nonparametric Envelopment of Data, StoNED）。该框架实现了生产技术的非参数估计与误差成分的统计分解，为效率分析提供了更为严谨的计量经济学基础。

    10.1 单产出模型设定¶

    首先，我们先研究多投入、单产出的情形，与SFA模型进行比较，以便理解二者的差异。多投入、多产出的情形以及多投入、多产出且有非期望产出的情形是单产出模型的扩展，我们在本节后接着讨论。

    首先，假设生产技术由前沿生产函数$f(\bold{x})$表示，其中$\bold{x}$是一个m维的投入向量。前沿$f(\bold{x})$表示用投入$\bold{x}$可以生产的最大产出，因此，函数$f(\bold{x})$表征了生产可能性集的边界。我们假设函数$f(\bold{x})$属于连续的、单调递增的、全局凹的函数类，可以是不可微的（我们把这个类表示为$F_2$。这相当于说明生产可能性集满足自由可处置性和凸性的经典DEA假设。与SFA不同的是，没有假定$f(\bold{x})$的具体函数形式。

    由于效率低下和噪声的影响，DMU$i$ 的被观察到的产出 $y_i$ 可能与 $f(\bold{x_i})$ 不同。我们遵循SFA文献，引入一个综合误差项 $\varepsilon=v_i-u_i$，它由非效率项$u_i$ 和随机噪声项$v_i$组成，形式为 $$ y_{i}=f(\bold{x_i}) +{\varepsilon_{i}}\ =f(\bold{x_i}) +v_i-u_i, \quad i=1, \ldots, n \tag{10.1} $$

    式中，变量 $u_i$ 和 $v_i$ 是随机变量，假定 $u_i$ 和 $v_i$ 相互独立，且 $u_i$ 和 $v_i$ 都与投入 $\bold{x}$ 无关。另外，假定 $ E\left(u_{i}\right)=\mu>0$，$Var\left(u_{i}\right)=\sigma^2_u<\infty$，且$ E\left(v_{i}\right)=0 $，$Var\left(u_{i}\right)=\sigma^2_v<\infty$。假设$\sigma^2_u$和$\sigma^2_v$在不同DMU间是恒定的，这被称为同方差性；具有异方差性的非效率和噪声模型需要对$u_i$和$v_i$的分布进一步施加假设，这里不做讨论，可见 Kuosmanen (2008) 3 的进一步讨论。在这个模型中，我们没有对 $f(\bold{x_i})$ 的具体函数形式施加限制，这与DEA文献的方法相同。另外，我们假设综合误差项 $\varepsilon_{i}$ 是由非效率项 $u_i$ 和随机噪声项$v_i$组成的，这与SFA文献的方法相同。因此，这个模型结合了DEA和SFA方法的优点。然而，与DEA模型相比，这个模型对非效率项 $u_i$ 的分布施加了假设。另外，与SFA模型相比，对 $u_i$ 和 $v_i$ 的分布的假设可能无法进一步精细化。

    除了假设综合误差项 $\varepsilon_{i}=v_i-u_i$ 外，我们可以施加一个替代性的假设：$\varepsilon_{i}=-u_i$，从而与DEA模型的假设保持相似。需要注意的是，这里的非效率项$u_i$不是最终需要的非效率，非效率的计算方法为$u_i$除以前沿（$y_i+u_i$）。另外，我们可以施加另外的替代性假设：$\varepsilon_{i}=v_i$，从而与凸非参数最小二乘法的假设相同。下面，我们一一介绍这些模型。

    对于模型（10.3），在样本中增加一个新的DMU会使未知参数的数量增加m+2（m为投入的维度），而Afriat不等式约束的数量增加2n（n为DMU的个数）。引入一个额外的输入变量会使未知参数的数量增加n，但对约束的数量没有影响。由于这些原因，标准的QP算法不足以处理有几百或几千个观测值的大样本。我们可以使用基于切割平面的算法来求解模型（10.3）。现在我们在下面解释该算法的各个步骤。

    切割面算法是解决大规模优化问题的有效工具，这些问题的约束条件数量非常多。在我们继续之前，我们定义一些在大规模优化文献中常用的术语。在这里，主问题指的是具有n(n-1)个约束条件的主问题(10.3)，而减少的主问题指的是具有相同目标和变量的问题，但约束条件只是主问题的约束条件的子集。这些方法的主要思想是开始解决最初有几个约束条件的问题--最初的减少的主问题。然后，我们找到被违反的约束条件，并以延迟的方式迭代添加它们--在每次迭代中，我们解决一个减少的主问题（但约束条件逐渐多于最初的减少的主问题）。因此，这种方法在大规模优化文献中也被称为延迟约束生成（Bertsimas and Tsitsiklis 1997）。这种方法的成功在很大程度上取决于寻找被违反的约束条件的效率。

    我们从n(n-1)个约束中的一小部分开始--一个初始的减少的主问题。通常情况下，只有n(n-1)个约束中的一小部分在主问题的最优解下是有效的，而且只用这些有效的约束来解决这个问题显然等同于解决完整问题。然而，这些有效的约束条件是事先不知道的。从一个 "接近 "有效约束集的约束集开始，一个关键的好处是，它可以大大减少在以后的迭代中增加的切割数量，并减少净计算负担。

    残差分解方法（method 参数）¶

    以下 3 种方法用于将 CNLS 残差 $\hat{\varepsilon}$ 分解为技术无效项 $\hat{u}$ 和统计噪声 $\hat{v}$：

    method 值 全称 返回类型 说明
    RED_QLE Quasi-Likelihood Estimation — 准似然估计（推荐），通过最大化修正对数似然函数估计 $\lambda$、$\sigma_u$、$\sigma_v$
    RED_MOM Method of Moments — 矩方法，利用残差的二阶和三阶矩估计 $\sigma_u$、$\sigma_v$
    RED_KDE Kernel Deconvolution Estimation — 核密度反卷积估计，使用 Richardson-Lucy 盲反卷积从残差中恢复 $\hat{u}$

    以上 3 种方法不直接返回效率值，而是设置 StoNED 对象的内部属性（sigma_u, sigma_v, mu），供效率计算方法使用。

    StoNED 效率获取方法¶

    CNLSSDweak + get_technical_efficiency(method)（Shephard 距离函数）¶
    返回项 类型 含义
    返回值 numpy array 技术效率 $TE = e^{-\hat{u}}$，每个 DMU 一个标量值

    适用于 CNLSSDweak（cet=CET_MULT）。$\hat{u}$ 通过 JLLMS 估计量获得，$TE \in (0, 1]$，越接近 1 效率越高。

    CNLSDDFweak + get_technical_efficiency_ratio(method) — gy 方向（期望产出扩张）¶
    返回项 类型 含义
    返回值 numpy array 技术效率 $TE = \dfrac{y}{y + \hat{u}}$，其中 $y$ 为期望产出

    当 gy ≥ 1 且 gx = 0、gb = 0 时使用。$\hat{u} > 0$ 表示产出扩张空间，$TE \in (0, 1]$。

    CNLSDDFweak + get_technical_efficiency_ratio(method) — gx 方向（投入缩减）¶
    返回项 类型 含义
    返回值 numpy array 技术效率 $TE = \dfrac{-x + \hat{\varepsilon}}{-x + \hat{\varepsilon} + \hat{u}}$，其中 $x$ 为投入

    当 gx ≥ 1 且 gy = 0、gb = 0 时使用。$\hat{u} > 0$ 表示投入缩减空间，$TE \in (0, 1]$。

    CNLSDDFweak + get_technical_efficiency_ratio(method) — gb 方向（非期望产出缩减）¶
    返回项 类型 含义
    返回值 numpy array 技术效率 $TE = \dfrac{-b + \hat{\varepsilon}}{-b + \hat{\varepsilon} + \hat{u}}$，其中 $b$ 为非期望产出

    当 gb ≥ 1 且 gy = 0、gx = 0 时使用。$\hat{u} > 0$ 表示非期望产出缩减空间，$TE \in (0, 1]$。

    CNLSSDweak vs CNLSDDFweak 对比¶

    特征 CNLSSDweak CNLSDDFweak
    距离函数类型 Shephard 距离函数 方向距离函数
    cet 参数 必须 CET_MULT（乘法复合误差） 无（内部 CET_ADDI，加法）
    求解器 "knitro"（非线性） "mosek"（线性）
    StoNED 效率方法 get_technical_efficiency() get_technical_efficiency_ratio()
    TE 公式 $TE = e^{-\hat{u}}$ $TE = \dfrac{w}{w + \hat{u}}$，$w$ 随方向变化（$y$/$x$/$b$）
    返回类型 numpy array numpy array
    适用方向 无方向概念 gy（产出）、gx（投入）、gb（非期望产出）

    4 种组合汇总¶

    模型 StoNED 方法 方向条件 TE 公式
    CNLSSDweak get_technical_efficiency() — $e^{-\hat{u}}$
    CNLSDDFweak get_technical_efficiency_ratio() gy ≥ 1 $y / (y + \hat{u})$
    CNLSDDFweak get_technical_efficiency_ratio() gx ≥ 1 $(x + \hat{\varepsilon}) / (x + \hat{\varepsilon} + \hat{u})$
    CNLSDDFweak get_technical_efficiency_ratio() gb ≥ 1 $(b + \hat{\varepsilon}) / (b + \hat{\varepsilon} + \hat{u})$

    CNLSSDweak 的 3 种 method（QLE/MOM/KDE）均返回相同的 numpy array 结构，差异仅在 $\hat{u}$ 的估计方法。CNLSDDFweak 的 3 种方向返回相同的 numpy array 结构，差异仅在 $TE$ 公式中的 $w$（基准值）。

    10.7 本章总结¶

    本章系统探讨了效率分析中非参数方法的前沿进展及其整合框架。传统方法中，数据包络分析（DEA）虽能避免参数设定却忽视随机噪声，随机前沿分析（SFA）虽可分解误差却依赖先验函数形式。凸非参数最小二乘法（CNLS）通过融合线性规划与二次规划技术，构建满足凸性、单调性公理的非参数前沿，既继承了DEA的模型灵活性，又引入SFA的统计推断框架。在此基础上，随机非参数数据包络（StoNED）通过误差项的概率建模与准似然估计，实现了生产技术的非参数估计与效率成分的统计分离，为效率分析提供了兼具严谨性与适应性的统一范式。

    研究进一步扩展至多投入多产出系统，通过方向距离函数（DDF）建模环境效率，构建包含非期望产出的随机前沿模型，并引入分位数回归增强对异方差数据的适应性。针对大样本计算难题，切割平面算法通过动态生成有效约束显著提升求解效率。以中国省级碳排放效率为案例的实证表明，StoNED框架能有效识别环境效率的动态演变，其CRS与VRS模型的结果差异揭示了规模效应对效率评估的系统性影响。该方法不仅弥补了传统DEA对数据噪声敏感的缺陷，更通过引入方向向量与背景变量拓展了政策分析的维度，为复杂系统效率评估提供了新的方法论工具。
